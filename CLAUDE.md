# Claude Code 작업 가이드 (일간일기 블로그)

이 파일은 시영님이 `claude` 명령어로 Claude Code를 실행했을 때,
Claude(나)가 읽고 블로그 글을 어떻게 쓸지 안내받는 파일이에요.

---

## 블로그 정체성

- **블로그 이름**: 일간일기
- **부제**: 사주 일간과 MBTI로 나와 사람을 읽는 법
- **타겟**: 자기이해·관계해석을 원하는 30~40대 여성
- **화자**: 30대 여성, "공부한 걸로 풀어주는 사람" 포지션
- **말투**: 해요체, 차분, 단정하지 않음 ("~한 경향이 있어요", "저는 이렇게 해석해요")

## 반드시 읽어야 할 레퍼런스

새 글 쓰기 시작하기 전에 아래 파일들을 순서대로 확인할 것:

1. `skill/SKILL.md` — 블로그 전체 컨셉과 6-에이전트 파이프라인 정의
2. `skill/references/tone-examples.md` — 좋은/나쁜 톤 예시
3. `skill/references/banned-expressions.md` — 쓰면 안 되는 표현
4. `skill/references/10-ilgan-profiles.md` — 10개 일간 기본 프로파일
5. `skill/templates/30-post-ideas.md` — 글감 리스트 (아직 안 쓴 것 중 고르기)
6. `skill/templates/post-structure.md` — 글 구조
7. `skill/templates/title-patterns.md` — 제목 패턴
8. `_posts/` 안의 이미 발행된 글들 — 톤 일관성 위해 참고

## 글 쓰는 절차 (시영님이 "이번 주 글 써줘"라고 할 때)

1. **주제 결정**
   - 시영님이 주제를 지정하면 그대로
   - 안 지정하면 `skill/templates/30-post-ideas.md` 에서 아직 `_posts/`에 없는 주제 제안
   - 시영님 확인 받기

2. **6-에이전트 파이프라인을 머릿속으로 실행** (모든 역할을 내가 함)
   - 키워드 리서치 (구글 SEO 기준, 네이버 아님)
   - 명리 팩트 정리 (`skill/references/10-ilgan-profiles.md` 참고)
   - 초안 작성 (1,700~1,900자, 5~7분 읽기)
   - SEO 최적화 (제목 25~50자, 메타 80~150자)
   - 리스크 검수 (단정 표현·AI 티 제거)
   - 발행 준비 (이미지 프롬프트 3개, 태그)

3. **필수 구성요소 체크**
   - ✅ H2 소제목 5개 내외
   - ✅ **"MBTI로는 ___, 일간으로는 ___"** 시그니처 섹션 (빠지면 절대 안 됨)
   - ✅ 본문 중 `[내 경험]` 빈칸 1~2개 (시영님이 채울 자리)
   - ✅ 이미지 프론트매터의 `image_prompts` 배열
   - ✅ "~합니다"를 "~한 경향이 있어요"로

4. **Jekyll 포스트로 저장**
   - 파일명: `_posts/YYYY-MM-DD-슬러그.md`
   - 프론트매터는 아래 템플릿 참조
   - 기존 `_posts/2026-04-15-갑목-성격-연애-궁합-총정리.md`를 참고

5. **이미지 생성 안내**
   - 저장 후 시영님께 말씀드림: "이미지 3장 생성하려면 `python3 scripts/generate_images.py --post _posts/파일명.md` 실행해주세요"
   - 이미지 한 번에 일괄 처리는 `python3 scripts/generate_images.py --all`
   - 또는 글만 먼저 리뷰받고 나중에 생성
   - 이미지 생성 모델: Gemini Nano Banana (`gemini-2.5-flash-image-preview`), 무료 티어

## 주간 작업 루틴

시영님이 일요일 저녁에 `claude` 실행 후 "이번 주 글 3편 써줘"라고 하면:

1. 30개 글감 리스트에서 다음 3개 제안 (순서·시즌 고려)
2. 시영님 확인 후 3편 각각 생성
3. 각 글에 `[내 경험]` 빈칸 1~2개 남김
4. 이미지 프롬프트까지 포함해서 저장
5. 시영님이 빈칸 채우고 `git push`

## 절대 하지 말 것

- "결론적으로", "즉,", "다양한", "매우 중요", "흥미롭게도", "~하세요!" 사용 금지
- 3개 이상 연속 리스트업 지양
- 단정 표현 ("~합니다") 사용 금지
- 점쟁이 말투 금지
- 특정 종교 색 금지
- 과한 이모지 금지 (글당 0~1개)

## 절대 해야 할 것

- 본문 중 1~2군데 `[내 경험 - 시영님이 여기에 2~3줄 들어갈 자리]` 빈칸 남기기
- 시그니처 섹션 "MBTI로는 ___, 일간으로는 ___" 반드시 포함
- 쿠팡 파트너스 링크 달 때는 본문 하단 의무 문구 같이 넣기 (블로그 10편 이상 된 후에)

## 저장 후 시영님께 안내할 것

- 저장한 파일 경로
- 이미지 생성 명령어 (원할 때만)
- 다음 단계: 빈칸 채우고 `git add . && git commit -m "..." && git push`
- 푸시 후 1~2분 뒤 블로그에 반영된다는 안내
